# Language Translation

语言翻译相关规则和工作流，尤其涉及将中文提示转换为英文以供下游工具使用。

## 输出纯度规则（关键）

**用户明确要求：** 翻译输出不得包含任何前言、解释、注释、引号、格式标记或周围对话上下文。输出必须是**纯净的翻译文本本身**，仅此而已。

### 禁止的内容（详列）

翻译输出严格禁止包含以下任何内容：

1. **前言或引言句** — 不得出现 `"Here is the translation:"`、`"Translated:"`、`"翻译如下:"`、`"英文翻译:"`、`"The English translation is:"` 等
2. **解释或评论** — 不得出现 `"I translated this as..."`、`"Note that..."`、`"This means..."` 等
3. **引号** — 不得使用 `"..."`、`'...'`、`「...」` 包裹翻译文本
4. **格式标记** — 不得出现 `[EN]`、`[ENGLISH]`、`[翻译]`、`---` 等标签
5. **后续问题或总结** — 不得出现 `"Does this work?"`、`"Let me know if..."`、`"Hope this helps"` 等
6. **礼貌前缀或客套话** — 不得出现 `"Sure!"`、`"Of course!"`、`"好的"` 等
7. **语种标识** — 不得标记输出文本的语种

### 唯一允许的输出

```
<仅此一行纯翻译文本，无任何附加内容>
```

这就够了。没有前言、没有解释、没有引号、没有格式标记、没有后续问题。

### 适用范围

- 此规则适用于所有中文→英文的翻译场景
- 同样适用于在调用 Codex 等外部工具前的中文→英文翻译中间步骤
- **优先级：** 此规则优先级高于任何其他响应格式偏好

### 执行检查清单

输出前检查：
- [ ] 没有 "Here is" / "Translated" / "翻译" 等引导词
- [ ] 没有引号包裹
- [ ] 没有括号说明
- [ ] 没有 `[EN]` 等标签
- [ ] 没有礼貌前缀
- [ ] 没有后续问题
- [ ] 输出是一个单一的文本块

## 使用场景

当用户给出中文提示且需要调用 Codex 或类似需要英文输入的工 Read时，先将提示翻译成英文，然后 *仅输出翻译结果* —— 不附加其他语言或评论。
