# Language Translation

语言翻译相关规则和工作流，尤其涉及将中文提示转换为英文以供下游工具使用。

## 输出纯度规则（关键）

- **输出纯度：** 当执行翻译时，输出 **仅** 包含翻译后的文本。
- 不得包含任何前言、解释、评论、引号或周围对话上下文。
- 用户已明确拒绝任何附带元内容的翻译。
- 这条规则同样适用于在调用 Codex 等外部工具前所需的中文→英文翻译中间步骤。

### Pitfalls — 沉默执行规则

- **静默执行。** 输出 _只有_ 翻译后的文本。不得含：
  - 引言句（如 `"Here is the translation:"`、`"Translated:"`）
  - 总结或后续问题
  - 引号包裹原文
  - 括号说明
  - 格式提示（如 `[EN]` / `[中文]`）
  - 礼貌前缀
  - 任何语种标识标签

- 用户对此有清晰且反复的偏好记录。**成功的响应是一个单一的文本块**——没有多余空白，没有附加注释或元内容。
- **歧义时：** 如果输出有歧义风险，不要主动添加说明。等待用户要求澄清。
- 此规则前置性高于其他任何引导或解释性的响应偏好。

## 使用场景

当用户给出中文提示且需要调用 Codex 或类似需要英文输入的工 Read时，先将提示翻译成英文，然后 *仅输出翻译结果* —— 不附加其他语言或评论。
