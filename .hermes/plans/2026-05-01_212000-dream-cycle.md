# Hermes "做梦" 记忆精炼机制 — 实现计划

> **For Hermes:** 使用 subagent-driven-development 按任务逐一实现。每个任务 2-5 分钟。

**目标:** 设计并实现一个后台定时运行的"做梦"流水线，自动扫描新会话、提取重要信息、合并到持久记忆文件 (MEMORY.md / USER.md)，并管理字符预算和矛盾解决。

**架构:** 纯 Python 脚本 `~/.hermes/scripts/dream_cycle.py`，被 cron 定时调用（每日凌晨）。LLM 交互通过 `hermes chat -q` 或直接调用 API（DeepSeek V4）完成。脚本是**有状态的**——维护一个标记文件记录上次处理到哪个 session，避免重复劳动。

**关键设计选择:**
- **LLM 只用在必须推理的地方**（判断重要性、检测矛盾、转相对日期）。纯机械操作（grep session、读文件、字符管理）不走 LLM。
- **使用 Hermes 的 session_search FTS5 快速定位**，而不是反复加载大型 JSON 文件。
- **脚本自包含**，不依赖 Hermes 内部 API，只读文件系统 + 调用 `hermes chat -q`（或直接 API）。

## Task 1: 创建标记文件管理模块

**目标:** 提供 `~/.hermes/memories/.dream_state.json` 的读写能力，记录"做梦"进度，避免重复扫描。

**文件:**
- 创建: `~/.hermes/scripts/dream_cycle.py`（先建骨架）

**设计:**

```json
{
  "last_processed_session": "20260501_211739_2439a368",
  "last_run": "2026-05-01T21:30:00+08:00",
  "total_runs": 12,
  "phases_executed": ["calibrate", "collect", "merge", "index"],
  "stats_summary": {
    "sessions_scanned": 3,
    "fragments_collected": 2,
    "entries_added": 1,
    "entries_removed": 0,
    "contradictions_found": 0
  }
}
```

**关键行为:**
- 每次运行记录 `last_processed_session`（最新处理的 session ID）
- 下次运行只处理 `last_processed_session` 之后创建的 session
- 如果文件不存在则从最早 session 开始（首次全量跑）
- 写后原子 rename，避免损坏

**实现骨架:**

```python
#!/usr/bin/env python3
"""Dream Cycle — Memory consolidation pipeline.
Scans new sessions, extracts important fragments, merges into MEMORY.md/USER.md,
resolves contradictions, and prunes stale entries.

Phase flow: calibrate → collect → merge → index
"""

import json, os, re, glob, sys, subprocess
from pathlib import Path
from datetime import datetime, timezone, timedelta
from typing import List, Dict, Optional, Tuple

HERMES_HOME = Path.home() / ".hermes"
MEMORIES_DIR = HERMES_HOME / "memories"
SESSIONS_DIR = HERMES_HOME / "sessions"
STATE_FILE = MEMORIES_DIR / ".dream_state.json"
MEMORY_FILE = MEMORIES_DIR / "MEMORY.md"
USER_FILE = MEMORIES_DIR / "USER.md"

ENTRY_DELIMITER = "\n§\n"

# ---------- State management ----------

def load_state() -> dict:
    if STATE_FILE.exists():
        return json.loads(STATE_FILE.read_text())
    return {"last_processed_session": None, "last_run": None, "total_runs": 0}

def save_state(state: dict):
    STATE_FILE.parent.mkdir(parents=True, exist_ok=True)
    tmp = STATE_FILE.with_suffix(".tmp")
    tmp.write_text(json.dumps(state, ensure_ascii=False, indent=2))
    tmp.rename(STATE_FILE)

def get_unprocessed_sessions(state: dict) -> List[Path]:
    """Return session files newer than last_processed, sorted by date."""
    last_id = state.get("last_processed_session")
    sessions = sorted(
        SESSIONS_DIR.glob("session_*.json"),
        key=lambda p: p.stat().st_mtime
    )
    if last_id:
        sessions = [s for s in sessions if last_id not in s.name and s.stat().st_mtime > ...]
        # Simplified: just skip sessions we've seen
        seen = set()
        # More robust: parse session_start from JSON
        return _filter_newer_than(sessions, last_id)
    return sessions

def _filter_newer_than(sessions: List[Path], last_session_id: str) -> List[Path]:
    """Return sessions created after the last processed one."""
    result = []
    found = False
    for s in sorted(sessions, key=lambda p: p.stat().st_mtime):
        if last_session_id in s.name:
            found = True
            continue
        if found:
            result.append(s)
    # If last_session_id not found (possible after cleanup), return last 48h
    if not found:
        cutoff = datetime.now(timezone.utc) - timedelta(hours=48)
        for s in sessions:
            try:
                mtime = datetime.fromtimestamp(s.stat().st_mtime, tz=timezone.utc)
                if mtime > cutoff:
                    result.append(s)
            except:
                pass
    return result
```

## Task 2: Phase 1 — 标定位置 (Calibrate)

**目标:** 读取当前 MEMORY.md 和 USER.md，建立已知条目索引，确定本次处理的 session 范围。

**逻辑:**
1. 读取 MEMORY.md → 解析为 `known_memory_entries: List[str]`（按 § 分割）
2. 读取 USER.md → `known_user_entries: List[str]`
3. 构建**条目哈希索引**：`{hash(entry.strip()): entry}` 用于快速去重
4. 读取 `.dream_state.json` → 获取 `last_processed_session`
5. 扫描 `~/.hermes/sessions/`，过滤出所有**未处理**的 session（按 `session_start` 比较）
6. 如果是首次运行（state 为空），限制处理最近 7 天的 session，避免 OOM

**输出:** `known_entries` + `pending_sessions: List[Path]`

**代码:**

```python
def phase1_calibrate(state: dict) -> dict:
    """Phase 1: Scan memory dir, build known index, find unprocessed sessions."""
    print("[DREAM] Phase 1: Calibrating...")
    
    # 1. Read existing memory files
    known = {"memory": _read_entries(MEMORY_FILE), "user": _read_entries(USER_FILE)}
    
    # 2. Build dedup index (stripped content hash)
    known_index = {}
    for target, entries in known.items():
        for e in entries:
            key = _entry_key(e)
            known_index.setdefault(target, {})[key] = e
    
    # 3. Find pending sessions
    last_id = state.get("last_processed_session")
    pending = []
    for spath in sorted(SESSIONS_DIR.glob("session_*.json"), key=lambda p: p.stat().st_mtime):
        if last_id and last_id in spath.name:
            pending = []  # reset, start collecting after this one
            continue
        if last_id:
            pending.append(spath)
        else:
            pending.append(spath)  # first run: all
    
    # 4. First run guard: cap at 7 days
    if not last_id and len(pending) > 20:
        cutoff = datetime.now(timezone.utc) - timedelta(days=7)
        pending = [p for p in pending if _get_session_mtime(p) > cutoff]
    
    return {"known": known, "known_index": known_index, "pending": pending}

def _read_entries(path: Path) -> List[str]:
    if not path.exists():
        return []
    raw = path.read_text(encoding="utf-8").strip()
    if not raw:
        return []
    return [e.strip() for e in raw.split(ENTRY_DELIMITER) if e.strip()]

def _entry_key(entry: str) -> str:
    """Normalize for dedup: lowercase, strip whitespace, remove common fillers."""
    return re.sub(r'\s+', ' ', entry.lower().strip())
```

## Task 3: Phase 2 — 收集 (Collect)

**目标:** 扫描未处理 session，提取"可能重要的片段"，检测与现有记忆的矛盾。

**策略分层（性能关键）：**

### 3a. 快速预筛（纯文本匹配，不走 LLM）

对每个 session 的 JSON，只读 **user 消息**（`role == "user"`）的前 500 字符，用正则匹配**高价值模式**：

```python
IMPORTANT_PATTERNS = [
    # 用户纠正
    (r'(错了|不是|不对|错误|纠正|正确.*是|实际.*是|改掉|不要再|别再用)', 'correction'),
    # 偏好声明
    (r'(我喜欢|我习惯|我偏好|prefer|不要|需要|要求|建议|最好|always use)', 'preference'),
    # 配置/环境
    (r'(配置|安装|设置|config|setup|环境|版本|路径|path|地址|端口|密码)', 'config'),
    # 记忆写入信号
    (r'(记住|保存|存到|写入|memory|技能|skill|记下来)', 'memory_signal'),
    # 项目专用
    (r'(项目|目录|代码|deploy|git|PR|commit|分支|branch|merge)', 'project'),
    # 日期/时间相关
    (r'(昨天|今天|明天|上周|下个月|几号|星期|deadline)', 'temporal'),
]
```

### 3b. 上下文采集

匹配到模式后，提取命中行前后各 2 条消息（user + assistant）作为上下文，控制每条消息 ≤ 800 字符。

### 3c. LLM 精筛（仅对预筛命中的片段）

对每个候选片段调用一次 LLM，判断：
- 是否是**值得记住的事实**（用户偏好、环境配置、项目约束）
- 如果是，提取**规范化的陈述句**
- 检测与 `known_index` 中现有条目是否**矛盾**

**LLM 提示设计:**

```
你是一个记忆精炼助手。分析以下对话片段，判断是否有值得永久记住的信息。

已知现有记忆：
{existing_entries}

对话片段：
{context}

请以 JSON 格式回答：
{
  "is_important": true/false,
  "target": "memory" 或 "user",
  "extracted_fact": "规范化的陈述句",
  "confidence": "high/medium/low",
  "contradicts_entry": "如果有矛盾，写被矛盾的旧条目原文（否则 null）",
  "contradiction_type": "correction/update/ambiguous/null",
  "reason": "为什么这个信息重要"
}
```

**注意:** LLM 调用通过 `hermes chat -q` 或直接 HTTP API。由于这是后台任务，需要静默执行。

### 3d. 矛盾检测

对比 `extracted_fact` 与 `known_index` 中所有条目：
- 如果 LLM 报告 `contradicts_entry != null` → 标记为**矛盾对**
- 矛盾类型：`correction`（用户说上次错了）→ 旧条目应被**替换**；`update`（信息更新）→ 旧条目应被**更新**；`ambiguous` → 保留双方，加备注

**输出:** `fragments: List[Dict]` — 每个包含 `{target, fact, confidence, contradicts, contradiction_type, session_id, timestamp}`

### 4c. 字符预算管理

当前 MEMORY.md 使用率 97% (2147/2200)，USER.md 49%。**做梦必须包含淘汰逻辑：**

```python
def _enforce_budget(entries, limit):
    """If total > limit, drop low-priority entries.
    Priority heuristic (ascending order of droppability):
    1. User corrections (highest)
    2. Active project configs
    3. Environment facts
    4. Lessons learned / tool quirks
    5. Old or superseded info (lowest)
    """
    total = len(ENTRY_DELIMITER.join(entries))
    if total <= limit:
        return entries
    
    # Sort entries: drop candidates first (shorter, older-looking)
    # Mark entries containing '2025' or '之前的版本' as stale
    stale_keywords = ['2025', '旧版本', '之前的', 'legacy', 'old', 'deprecated']
    
    def priority(e):
        score = 0
        for kw in stale_keywords:
            if kw in e:
                score -= 1  # lower priority = more droppable
        score -= len(e) * 0.001  # slightly prefer conciseness
        return score
    
    sorted_entries = sorted(entries, key=priority)
    
    # Remove from front until under budget (with 5% headroom)
    while len(ENTRY_DELIMITER.join(sorted_entries)) > limit * 0.95 and len(sorted_entries) > 1:
        removed = sorted_entries.pop(0)
        print(f"  [BUDGET] Dropped entry ({len(removed)} chars): {removed[:80]}...")
    
    return sorted_entries
```

## Task 5: Phase 4 — 整理与索引 (Index)

**目标:** 最终写入 MEMORY.md/USER.md，清理过时条目，解决跨文件矛盾，更新状态文件。

### 5a. 跨文件矛盾解决

检查 MEMORY.md 和 USER.md 之间是否有冲突：
- 同一个事实在两个文件中有不同描述
- 对比方式：对两条目做简单关键词交集，如果重叠 > 70% 但内容不同，标记

处理规则：
- 冲突时，USER.md 优先（用户画像的权威性高于 agent 笔记）
- 或合并到 MEMORY.md，USER.md 只保留纯粹的用户偏好

### 5b. 过时条目清理

检测条件：
- 条目中包含 "2025" 年份且当前是 2026 — 标记为可能过时
- 条目被矛盾标记覆盖 — 删除旧条目
- 条目字符数很少（<20 chars）且无关键信息 — 合并到相邻条目

```python
def phase2_collect(pending_sessions, known_index):
    """Phase 2: Collect important fragments from new sessions."""
    fragments = []
    for spath in pending_sessions:
        try:
            session_data = json.loads(spath.read_text(encoding="utf-8"))
        except (json.JSONDecodeError, OSError) as e:
            print(f"  [WARN] Cannot parse {spath.name}: {e}")
            continue
        
        session_id = session_data.get("session_id", spath.stem)
        session_start = session_data.get("session_start", "")
        messages = session_data.get("messages", [])
        
        # Quick pre-filter: scan user messages for high-value patterns
        candidates = []
        for i, msg in enumerate(messages):
            if msg.get("role") != "user":
                continue
            content = str(msg.get("content", ""))
            if not content:
                continue
            
            # Check against important patterns
            for pattern, category in IMPORTANT_PATTERNS:
                if re.search(pattern, content):
                    # Collect context: 2 before, 2 after
                    start_idx = max(0, i - 2)
                    end_idx = min(len(messages), i + 3)
                    context_msgs = messages[start_idx:end_idx]
                    
                    # Truncate each message content
                    truncated = []
                    for cm in context_msgs:
                        c = str(cm.get("content", ""))
                        if len(c) > 600:
                            c = c[:600] + "..."
                        truncated.append({"role": cm.get("role"), "content": c})
                    
                    candidates.append({
                        "pattern": category,
                        "context": truncated,
                        "session_id": session_id,
                        "session_start": session_start,
                    })
                    break  # one match per user message is enough
        
        # LLM refine (only if candidates found)
        for cand in candidates:
            fact = _llm_extract_fact(cand, known_index)
            if fact:
                fact["session_start"] = session_start
                fragments.append(fact)
    
    # Sort by confidence
    fragments.sort(key=lambda f: {"high": 0, "medium": 1, "low": 2}.get(f.get("confidence"), 3))
    return fragments

        if m:
            try:
                resp = json.loads(m.group())
                if resp.get("is_important") and resp.get("extracted_fact"):
                    return resp
            except: pass
    return None


if __name__ == "__main__":
    main()
```

## 风险和注意事项

| 风险 | 缓解措施 |
|------|---------|
| **LLM 调用耗时**：每个候选片段一次 LLM 调用，如果 session 多可能很慢 | 预筛阶段严格把关，只放行匹配模式的片段；LLM 调用设 60s 超时；首跑限 7 天数据 |
| **字符预算溢出**：MEMORY.md 已 97% | Phase 3 必须有淘汰逻辑，不能只加不减 |
| **并发写入冲突**：cron 运行时用户也在聊天写入 memory | 使用文件锁（fcntl）保护 MEMORY.md 写入；原子 rename |
| **误删重要记忆** | 淘汰时只删**低优先级 + 含陈旧关键词**的条目；保留所有 correction 类条目 |
| **首次运行处理大量 session** | ceil 为 7 天或 20 个 session，先处理最近的 |
| **hermes chat -q 在 cron 中不可用** | 备用方案：直接 HTTP POST 到 DeepSeek API |
